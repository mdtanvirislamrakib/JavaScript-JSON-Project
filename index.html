Frequently Asked Questions

The difference between var, let, and const
var is function-scoped and allows re-declaration. let is block-scoped and can be reassigned but not re-declared. const is also block-scoped but cannot be reassigned. var hoists with undefined, while let and const hoist but remain in the temporal dead zone until initialized.
The difference between map(), forEach(), and filter()
map() returns a new array by transforming each element. forEach() loops through elements but doesn’t return a new array. filter() creates a new array with elements that meet a condition. Use map() for transformation, forEach() for side effects, and filter() to extract specific elements.
Explain arrow functions and how they are different from regular functions
Arrow functions have a shorter syntax and inherit this from the surrounding scope, unlike regular functions that define their own this. Arrow functions cannot be used as constructors, don't have the arguments object, and can't be called with new. Regular functions can be more flexible in certain cases.
How JavaScript Promises work
JavaScript Promises represent asynchronous operations. They start in the pending state, then transition to fulfilled if successful or rejected if an error occurs. You handle results with .then() for success and .catch() for errors. Promises allow chaining and managing asynchronous flows more effectively than callbacks.
How closures work in JavaScript
Closures in JavaScript occur when a function retains access to its lexical scope, even after the outer function has finished executing. This means inner functions can access variables from their outer functions, allowing for data encapsulation and private variables. Closures are key for creating function factories and callbacks.
